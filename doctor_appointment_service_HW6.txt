Необходимо спроектировать сервис записи на приём к доктору:
  а. Доменная модель (клиника, пациент, доктор, вид услуги, расписание, ..), в виде текста. Домен - атрибуты
  б. Сценарии (выбор доктора, выбор времени, запись, перенос, отмена, приём, ..), в виде Актор - Прецедент (из первой лекции)
  в. Адаптеры интерфейсов в виде компонент (подключение к БД, шлюзы к интерфейсу, шлюзы к третьим источникам)
  г. Конкретные технологии (БД, фреймворки UX/UI, брокеры) - нужна помощь преподавателя, т.к. это темы будущих лекций

ДЗ: самостоятельно доработать схему, чтобы можно было осуществить следующие сценарии: выбор доктора, выбор времени, запись, перенос, отмена, приём
-------------------------------------------------------------

Моя схема:
https://dbdesigner.page.link/MqnJU4z7ZW6pjhAx7

Сценарии в диаграмме UML

1. Выбор доктора:
Авторизоваться -> открыть перечень услуг -> выбрать услугу -> выбрать доктора

2. Выбор времени:
Авторизоваться -> открыть перечень услуг -> выбрать услугу -> выбрать доктора -> выбрать время приёма

3. Запись на приём:
Авторизоваться -> открыть перечень услуг -> выбрать услугу -> выбрать доктора -> выбрать время приёма -> подтвердить запись на приём

4. Перенос записи на приём:
Авторизоваться -> открыть журнал предварительной записи -> выбрать запись на услугу -> отменить запись -> выбрать время приёма -> подтвердить запись на приём

5. Отмена записи на приём:
Авторизоваться -> открыть журнал предварительной записи -> выбрать запись на услугу -> отменить запись -> подтвердить отмену записи на приём (recording_reception: false)

6. Отметка о состоявшемся приёме:
Авторизоваться -> открыть журнал регистрации -> выбрать запись на услугу -> просмотреть статус, дату, время и результат приёма
