Урок 5. Горизонтальные уровни и вертикальные срезы архитектуры (семинар). ДЗ

Предложить варианты связывания всех уровней для "Автомобильный навигатор с картами" – сценарии использования. 3-4 сценария. Добавить недостающие компоненты для этих сценариев для "Автомобильный навигатор с картами".

1. Что видит пользователь?
  1.1. Карта
  1.2. Поле ввода адреса
  1.3. Меню
  1.4. Местоположение
2. Какие задачи можно решать (делать)?
  2.1. Показать своё местоположение
  2.2. Ввести (найти) адрес
  2.3. Построить маршрут
  2.4. Запустить навигацию
  2.5. Озвучивание навигации
3. Бизнес-логика не характерная для приложения
  3.1. Авторизация пользователя
  3.2. Запрос по API о дорожной ситуации
  3.3. Запрос со спутников об определении местоположения
4.  Какие и где хранятся данные?
  4.1. В памяти навигатора загруженные карты (flash)
  4.2. В БД в таблицах (история поиска, избранные адреса)
  4.3. API (интернет)
  4.4. ГЛОНАСС

5. Какие уровни архитектуры будут?
  5.1. Графический интерфейс
    5.1.1 UX
    5.1.2 UI
  5.2. Бизнес-логика характерная для приложения:
    4.2.1. Определение местоположения
    4.2.2. Навигация
  5.3. Бизнес-логика не зависящая от приложения
    5.3.1. Соединение со спутниками
    5.3.2. Запрос данных о дорожной ситуации
    5.3.3. Запрос в БД о пользователе
  5.4. Сохранение данных
    5.4.1. Сохранение в памяти навигатора (flash)
    5.4.2. Сохранение данных в БД о пользователе

Сценарии использования связывания всех уровней архитектуры:
1. Ввести адрес в поиск -> запустить навигацию -> озвучивание навигации -> получение информации о дорожной ситуации -> сохранить адрес в избранное
2. Открыть меню. Залогиниться -> авторизоваться -> загрузить карту местности -> определить своё местоположение -> посмотреть информацию о дорожной ситуации
3. Открыть меню -> выбрать загруженную карту -> определить своё местоположение -> найти адрес на карте -> построить маршрут -> запустить навигацию -> озвучивание навигации

Доменные функции
  1. Взаимодействие с БД
  2. Взаимодействие с пользователем
  3. Взаимодействие со спутниками
  4. Взаимодействие с сервером (API)

Функции системы
  1. Определение местоположения
  2. Поиск адреса
  3. Построение маршрута
  4. Навигация по маршруту
  5. Озвучивание навигации
  6. Показ информации о ситуации на дорогах
  7. Загрузка и сохранение карт для офлайн навигации
  8. Создание профиля
  9. Добавление адреса в избранное
  10. Редактирование профиля
