Урок 9. Способы организации передачи данных между компонентами приложения, протоколы и API. REST, gRPC, очереди (семинар). ДЗ

Задание

1. Необходимо спроектировать контракт API для приложения заказа (создания) ресурсов в облачном сервисе (CPU, RAM, ROM, IP адрес, OS)
  a. Доменная модель в виде текста Домен - атрибуты
  b. Сформировать REST запросы: GET, POST, PUT, DELETE. Компонент бизнес-логики проводит CRUD операции с БД, инструмент проектирования API контракта http://swagger.io
  c. Определить и передавать параметры в формате JSON (тип ресурса - хранилище или компьютер, процессор, ОЗУ, диск - объём и тип, ОС, IP адрес)
  d. Сгенерировать (автоматически на ресурсе http://swagger.io) код на разных языках программирования

ДЗ:

1. Добавить в swagger получение списка имеющихся процессоров (get) удаление процессора по идентификатору (delete)

2. Разработать экранные формы интерфейса для заказа ресурсов(CPU) в облачном сервисе в https://www.figma.com/ или https://app.diagrams.net/

3. Разработать полную ERD домена для всех ресурсов (CPU, RAM, ROM, IP адрес, OS) в https://www.dbdesigner.net/

4. Дополнить swagger ответами домена (сетевые ответы) о статусе заказа ресурсов (создан, ошибка, нет ответа) и смоделировать ошибки REST «400, 500» типов

5. Имплементировать сгенерированный swagger код в приложения студента

Свагер: https://editor.swagger.io/
